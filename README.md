# 🗑️ 212 宿舍倒垃圾自动化提醒系统 v2.0

### 项目初衷
我们宿舍（212）实行每天一人倒垃圾的轮流制度。但现实中，大家总是会陷入“今天该谁了？”的终极哲学思考。为了彻底解决这个困扰，我将原本的 1.0 脚本升级为了 **2.0 云端架构版**。它不仅能在线显示排班，还能每天准时在微信里“催债”，且支持数据动态调整。

---

### 核心功能
* **云端记忆中枢**：接入 **Supabase (PostgreSQL)** 云数据库，人员变更或日期调整只需在数据库后台修改，无需改动代码。
* **实时网页查询**：基于 Streamlit 部署的专属网页，支持动态日期查询，今日大元帅与明日接客候选人一目了然。
* **微信自动通知**：利用 GitHub Actions 每天早上 **7:05** 定时触发，通过 PushPlus 接口将提醒精准推送到全宿舍人的微信上。
* **企业级安全隐私**：所有 API Key 和数据库密码均托管于 GitHub/Streamlit Secrets 保险箱，隐私数据稳如老狗。

---

### 技术栈
* **语言**：Python 3.9 (Full-Stack)
* **数据库**：Supabase (Cloud PostgreSQL)
* **前端展示**：Streamlit (Web App)
* **自动化脚本**：GitHub Actions
* **消息推送**：PushPlus (WeChat Webhook)

---

### 📖 如何部署你自己的版本？
如果你（比如 314 宿舍的兄弟）也想“白嫖”这套系统，只需以下几步：
1. **Fork 本仓库**：点击右上角 Fork 到你自己的账号下。
2. **准备数据库**：在 [Supabase](https://supabase.com/) 新建项目，创建一个名为 `dorm_rules` 的表（记得关闭 RLS 安全限制），并填入你宿舍的信息。
3. **配置保险箱**：在 GitHub 仓库的 `Settings -> Secrets` 中存入你的 `SUPABASE_URL`、`SUPABASE_KEY` 和 `PUSHPLUS_TOKEN`。
4. **修改宿舍号**：将 `app.py` 和 `notify.py` 中的 `dorm_id` 改成你自己的宿舍号。
5. **开启闹钟**：在 GitHub Actions 页面启用并手动运行一次 `Daily Trash Reminder` 即可。

---

### 运行逻辑
系统不再使用硬编码，而是采用**动态拉取**模式：
机器人每天早晨准时从 GitHub Actions 中醒来，跨越机房访问位于新加坡的 Supabase 数据库，提取配置信息。程序会自动计算当前日期与数据库中起始日期的天数差，并按人员名单取模循环，确保公平公正公开。
