# 🗑️ 212 宿舍倒垃圾自动化提醒系统 v3.0

### 项目初衷

我们宿舍（212）实行每天一人倒垃圾的轮流制度。但现实中，大家总是会陷入“今天该谁了？”的终极哲学思考。为了彻底解决这个困扰，我将原本的 1.0 脚本升级为了 **2.0 云端架构版**。它不仅能在线显示排班，还能每天准时在微信里“催债”，且支持数据动态调整。

---

### 🚀 项目重大更新：迈向全栈管理 (v3.0)
在 2.0 时代，我们需要去数据库后台修改数据；而 **3.0 版本** 彻底实现了“管理自给自足”。我们新增了**前端管理控制台**，现在你可以直接在网页上通过密码验证，实时修改舍友名单和排班规则。

这不仅是排班表，这是一个具备完整权限控制、云端存储、自动提醒功能的微型 SaaS 系统。

---

### ✨ 3.0 核心功能
* **🔐 网页管理控制台**：新增 Admin Panel，支持输入管理口令解锁。无需登录数据库后台，直接在手机网页端修改数据。
* **⚡ 实时热更新**：采用 `st.rerun()` 机制与缓存清理技术。在网页修改并同步后，排班结果秒级刷新，无感切换。
* **🛡️ 数据库安全防御**：通过 Supabase SDK 参数化查询，天然抵御 **SQL 注入**攻击。即使是 3.0 的动态写入模式，数据依然稳如泰山。
* **⏰ 微信全端同步**：GitHub Actions 闹钟每天 **07:05** 准时拉取最新云端配置。网页改一次，微信通知同步更新。

### 📖 快速部署 (3.0 升级指南)
如果你是 314 宿舍或其他宿舍的兄弟，请按照以下步骤部署：
1. **Fork 本仓库**：点击右上角 Fork 按钮。
2. **数据库准备**：在 [Supabase](https://supabase.com/) 创建表 `dorm_rules`，填入初始数据。
3. **配置保险箱 (Secrets)**：
   - **GitHub 侧**：设置 `SUPABASE_URL`、`SUPABASE_KEY`、`PUSHPLUS_TOKEN`。
   - **Streamlit 侧**：除上述两个外，额外增加 `ADMIN_PASSWORD` (你的管理后台口令)。
4. **修改宿舍号**：将代码中 `dorm_id` 改为你的宿舍号。
5. **开启定时任务**：在 Actions 页面启用并手动触发 `Daily Trash Reminder`。

---

### 🏗️ 运行逻辑
系统采用**双向数据流**架构：
1. **读取流**：网页与 GitHub 闹钟实时从新加坡数据库获取最新的排班规则。
2. **写入流**：管理员通过网页端加密验证后，直接覆写云端数据库，实现排班规则的动态热修正。
3. **计算层**：基于日期锚点自动取模，精准定位每日“倒垃圾大元帅”。
