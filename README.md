# 🗑️ 212 宿舍倒垃圾自动化提醒系统 v3.1.1

### 💡 项目初衷
我们宿舍（212）实行每天一人倒垃圾的轮流制度。为了彻底解决“今天该谁了？”的终极哲学思考，我将系统一路进化到了 **3.1.1 全栈旗舰版**。它不仅能在线显示排班，准时在微信里“催债”，如今还学会了“精准放假”和“排版美化”。

---

### 🚀 v3.1.1 重大更新：精细化运营与视觉重构
在 3.1 版本的节假日休眠基础上，v3.1.1 进行了深度细节打磨：
1. **精准节假日识别**：修复了原版会将“普通周末”误判为假期的 Bug。现在系统仅在法定大长假（如国庆、春节）休眠，普通周末照常提醒，确保宿舍卫生不留死角。
2. **UI 像素级优化**：注入自定义 CSS 魔法，解决了长标题在网页端换行的尴尬，实现 PC 端单行完美对齐，并自动适配手机端显示。

这已经是一个高度成熟、具备人文关怀、极致追求 UI 体验的微型 SaaS 系统。

---

### ✨ 核心功能
* **🔐 网页管理控制台**：新增 Admin Panel，支持输入管理口令解锁。支持在手机端实时修改舍友名单、调整初始日期与负责人。
* **🏖️ 精准休眠算法**：基于 `chinesecalendar` 库，智能识别“法定节假日”与“普通休息日”。只有大家确定回家的长假才会静默，普通双休依然是倒垃圾的日子。
* **🎨 响应式 UI 适配**：采用 CSS 注入技术，强制标题在不同尺寸屏幕下均能呈现最佳视觉比例，版本号 3.1.1 完美展示。
* **🛡️ 安全加固**：通过参数化查询抵御 SQL 注入；管理后台预留安全扩展接口，确保数据稳如泰山。
* **⏰ 微信全端同步**：GitHub Actions 闹钟每天 **07:05** 准时拉取最新云端配置，网页端任何修改均会实时反映在推送通知中。

---

### 🛠️ 技术栈
* **语言**：Python 3.9 (Full-Stack Logic)
* **数据库**：Supabase (Cloud PostgreSQL)
* **前端框架**：Streamlit (Interactive UI + CSS Injection)
* **自动化脚本**：GitHub Actions
* **通信协议**：PushPlus WeChat Webhook
* **算法依赖**：`chinesecalendar` (高精度中国历法库)

### 📖 快速部署指南
如果你是 314 宿舍或其他宿舍的兄弟，请按照以下步骤部署：
1. **Fork 本仓库**：点击右上角 Fork 按钮。
2. **依赖安装**：确保 `requirements.txt` 中包含 `chinesecalendar` 等所有 3.1.1 核心组件。
3. **数据库配置**：在 [Supabase](https://supabase.com/) 创建 `dorm_rules` 表并填入初始排班锚点。
4. **环境变量 (Secrets)**：
   - **GitHub**：设置 `SUPABASE_URL`、`SUPABASE_KEY`、`PUSHPLUS_TOKEN`。
   - **Streamlit**：额外增加 `ADMIN_PASSWORD` 作为你的网页管理口令。
5. **修改宿舍号**：将代码中所有的 `dorm_id` 改为你自己的宿舍号。

---

### 🏗️ 运行逻辑
系统采用**双向数据流 + 动态逻辑校验**架构：
1. **智能休眠网关**：闹钟触发后识别 `on_holiday` 与 `holiday_name`。若仅为周末则放行，若为法定节假日则静默。
2. **读取流**：网页与 GitHub 闹钟实时从新加坡数据库同步最新的排班规则。
3. **写入流**：管理员通过加密验证后，直接覆写云端数据库，实现排班规则的热修正。
4. **CSS 渲染层**：前端强制执行字体大小微调，确保长标题不换行。
