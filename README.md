# 🗑️ 212 宿舍倒垃圾自动化提醒系统 v3.1

### 💡 项目初衷
我们宿舍（212）实行每天一人倒垃圾的轮流制度。但现实中，大家总是会陷入“今天该谁了？”的终极哲学思考。为了彻底解决这个困扰，我将原本的脚本一路升级为了 **3.1 智能化全栈版**。它不仅能在线显示排班，准时在微信里“催债”，如今还学会了“看日子放假”。

---

### 🚀 3.1 版本重大更新：注入“人文关怀”
在 3.0 时代，我们实现了前端界面的“自给自足”。而 **3.1 版本** 重点优化了系统的用户体验：
* **节假日智能休眠**：机器没有感情，但架构师有。放假期间绝不打扰兄弟们睡懒觉。

这已经不是一个简单的排班表，而是一个具备完整权限控制、云端存储、节假日算法识别的微型 SaaS 系统。

---

### ✨ 核心功能
* **🔐 网页管理控制台**：新增 Admin Panel，支持输入管理口令解锁。无需登录数据库后台，直接在手机网页端修改数据。
* **🏖️ 节假日智能休眠**：接入 `chinesecalendar` 库，精准识别中国法定节假日与调休补班。放假期间闹钟自动静默，工作日（含补班）准时唤醒。
* **⚡ 实时热更新**：采用 `st.rerun()` 机制。在网页修改并同步后，排班结果秒级刷新，无感切换。
* **🛡️ 数据库安全防御**：通过 Supabase SDK 参数化查询，天然抵御 **SQL 注入**攻击。即使是动态写入模式，数据依然稳如泰山。
* **⏰ 微信全端同步**：GitHub Actions 闹钟每天 **07:05** 准时拉取最新云端配置。网页改一次，微信通知同步更新。

* **语言**：Python 3.9 (Full-Stack Logic)

### 📖 快速部署指南 (3.1 版本)
如果你是 314 宿舍或其他宿舍的兄弟，请按照以下步骤部署：
1. **Fork 本仓库**：点击右上角 Fork 按钮。
2. **环境依赖准备**：确保你的 `requirements.txt` 中包含 `chinesecalendar`、`supabase` 和 `streamlit` 等核心库。
3. **数据库准备**：在 [Supabase](https://supabase.com/) 创建表 `dorm_rules`，填入初始数据。
4. **配置保险箱 (Secrets)**：
   - **GitHub 侧**：设置 `SUPABASE_URL`、`SUPABASE_KEY`、`PUSHPLUS_TOKEN`。
   - **Streamlit 侧**：除上述两个外，额外增加 `ADMIN_PASSWORD` (你的管理后台口令)。
5. **修改宿舍号**：将 `app.py` 和 `notify.py` 代码中的 `dorm_id` 改为你的宿舍号。
6. **开启定时任务**：在 Actions 页面启用并手动触发 `Daily Trash Reminder` 验证运行状态。

### 🏗️ 运行逻辑
系统采用**双向数据流 + 智能拦截**架构：
1. **休眠网关（新增）**：GitHub 闹钟触发后，首先校验国家法定节假日历。若为假期则静默终止；若为工作日则放行。
2. **读取流**：网页与 GitHub 闹钟实时从新加坡数据库获取最新的排班规则。
3. **写入流**：管理员通过网页端加密验证后，直接覆写云端数据库，实现排班规则的热修正。
4. **计算层**：基于日期锚点自动取模，精准定位每日“倒垃圾大元帅”。
